# ФЛИН — Дизайн-спек

**Дата:** 2026-05-27  
**Статус:** Утверждён

---

## 1. Что строим

Семейное веб-приложение для ведения бюджета двух пользователей — Стаса и Любы. Главная ценность: внести расход одной фразой и сразу увидеть обновлённый остаток. AI помогает с парсингом текста, категоризацией и ответами на вопросы — но не с расчётами.

  prisma.ts                   ← Prisma client singleton
  budget.ts                   ← Детерминированные расчёты остатков

**Ключевой принцип:** `lib/budget.ts` считает деньги детерминированно. AI не участвует в расчётах — только в интерпретации текста и ответах на вопросы.

## 4. Навигация (5 разделов)

Нижняя панель на мобильном:

1. **Главная** — остатки + быстрый ввод
2. **Операции** — список с фильтрами
3. **План** — регулярные и будущие расходы
4. **ФЛИН** — AI-чат
5. **Настройки** — бюджетный период, пользователи, категории

---

## 5. Главный экран

Стиль: светлый, зелёный акцент, карточки с воздухом, мобильный first.

Элементы сверху вниз:
1. Шапка: логотип ФЛИН + имя текущего пользователя
2. Карточка остатка — градиент зелёный, крупные цифры, внутри — прогнозный остаток красным
3. Блок быстрого ввода — поле + кнопки «Добавить» и «Спросить»
4. Подсказка ФЛИНА — жёлтый блок, генерируется асинхронно (не блокирует загрузку страницы), кешируется на 1 час
5. Ближайшие платежи — 2–3 строки
6. Последние операции — 3–5 строк с иконкой категории

**Цветовые правила:**
- Зелёный — остаток в норме, основные действия
- Красный — риск, расходы
- Жёлтый — предупреждения и советы ФЛИНА

---

## 6. Авторизация

При первом открытии — экран выбора: «Я — Стас» или «Я — Люба». Выбор хранится в http-only cookie (`flin_user_id`) чтобы Server Actions могли читать userId без передачи в форме. Переключение — в настройках. Пароли не нужны.

Стас — admin (полный доступ), Люба — member (вносит расходы и доходы, видит все данные, задаёт вопросы AI).

## 7. Модели данных

Точно по `FLIN_CLAUDE_CODE_SPEC.md` раздел 12:

- `User` — id, name, role (admin/member), avatarColor
- `BudgetPeriod` — startDate, endDate, plannedBudget, startingBalance, includeRecurringInBudget
- `Transaction` — type (income/expense), amount, description, categoryId, userId, date, source (manual/ai), confidence
- `Category` — name, type (income/expense), color
- `RecurringPayment` — name, amount, categoryId, dayOfMonth, frequency, isActive
- `PlannedExpense` — name, estimatedAmount, priority, necessity, status
- `AiChatMessage` — role (user/assistant), content

---

## 8. Расчёты бюджета (детерминированные)

```
Текущий остаток = startingBalance + доходы − расходы
Фактический остаток = текущий − будущие регулярные платежи − запланированные расходы
Безопасный дневной лимит = фактический остаток / дней до конца периода
```

Если фактический остаток < 0 — показываем предупреждение с конкретной суммой.

---

## 9. AI-интеграция

| Сценарий | Модель | Метод |
|---|---|---|
| Парсинг расходов из текста | GPT-4o-mini | Server Action, structured JSON output |
| Совет дня на главной | GPT-4o-mini | Server-side при загрузке страницы |
| Ответы в чате | GPT-4o | API route со стримингом |

Системный промпт чата включает: текущий остаток, фактический остаток, список категорий, историю операций за период, регулярные платежи, запланированные расходы. AI отвечает коротко, с конкретными цифрами, без канцелярита.

- Тестовые операции: продукты 2 500 ₽, кофе 300 ₽, такси 1 200 ₽, аптека 540 ₽
